# Phase 01 — Foundation (Vite, React, Bun, App Shell)

**Prerequisites:** None.  
**Next phase:** [`02.md`](02.md) Guided learning engine.

**Context:** Establish the React + Vite application under `frontend/`, wire Bun scripts, apply base styling and layout so later phases plug in guided examples and playgrounds without refactoring the shell.

---

## Objectives

1. Create `frontend/` as a **Vite + React** project runnable with **Bun**.
2. Provide **app shell**: header, navigation area for topics (can be stub list), main content region, sidebar placeholder for future “Variables / Console / Tests”.
3. Establish **design tokens** (CSS variables): background, foreground, muted text, accents, borders, fonts (e.g. JetBrains Mono for code UI, readable sans for copy).
4. Add **routing or view state** for at least two modes tabs: **Learn** / **Practice** (Practice can be stub: “Coming in Phase 03”).
5. Replace reliance on CDN Tailwind from the prototype with **maintainable CSS** (could be Vanilla CSS Modules or scoped CSS imports).

---

## Tasks

### 1.1 Project scaffold

- [ ] Initialize Vite React + TypeScript in `frontend/` (or add `package.json`, `vite.config.ts`, `tsconfig.json`, `index.html`, `src/main.tsx`).
- [ ] Add `bun` scripts: `dev`, `build`, `preview`; document in README or root doc if needed.
- [ ] Configure path alias `@/` → `src/` if desired.
- [ ] `.gitignore` extended for `node_modules`, `dist`, Vite caches.

### 1.2 App shell layout

- [ ] Root component: structured layout (responsive): header, optional left rail for topics, main column, optional right sidebar.
- [ ] Tabs or segmented control: **Learn** | **Practice** (Practice content = placeholder panel).
- [ ] Stub topic list:** e.g. `Variables`, `Operators`, … as static constants in `src/data/stubTopics.ts` (minimal list; full curriculum in Phase 04).

### 1.3 Styling

- [ ] `src/styles/tokens.css`: `:root` variables for colors, spacing, radius, fonts.
- [ ] `src/styles/global.css`: reset/normalize-lite, base body styles, scrollbar if desired (match prototype feel without copying every line verbatim).
- [ ] Responsive breakpoints: collapsible sidebar on small widths.

### 1.4 Developer ergonomics

- [ ] ESLint + optional React hooks rules (minimal config).
- [ ] Confirm `bun run build` succeeds.

---

## Acceptance Criteria

| # | Criterion |
|---|-----------|
| A1 | `cd frontend && bun install && bun run dev` opens the app with no runtime errors. |
| A2 | Layout shows header + main area + placeholders suitable for sidebar. |
| A3 | **Learn** / **Practice** switch works (state only); Practice panel shows intentional placeholder referencing Phase 03. |
| A4 | At least ~5 stub topics appear in navigation (titles only). |
| A5 | Design tokens live in a dedicated CSS file; no scattered hard-coded hex everywhere in components. |
| A6 | `bun run build` completes without errors. |
