# Phase 02 — Guided Learning Engine

**Prerequisites:** Phase [`01.md`](01.md) complete.  
**Next phase:** [`03.md`](03.md) Playground.

**Context:** Phase 01 provides the shell. This phase implements the **curated step-based** code learner: code display with **line + optional segment** highlighting, variable/array visualization, educational console, explanation panel, playback controls, sound, speed, keyboard shortcuts. Port and structure data from [`frontend/init_example.html`](../../frontend/init_example.html) instead of keeping one monolithic HTML file.

---

## Objectives

1. Define **TypeScript types** for topics, examples, code lines, segments, and execution steps.
2. Implement **CodeViewer** that renders lines from structured segments (not one raw string per line only) so partial-line highlight is possible.
3. Implement **playback** state machine: step index, play/pause, timer for auto-advance, step forward/back, reset, progress.
4. Implement **VariableInspector** (primitives + arrays with index highlight from step metadata); objects can be simple key-value list in v1.
5. Implement **ConsolePanel** showing cumulative `console.log` output for the current example run state.
6. Port **SoundEngine** (Web Audio API) behind a hook; respect mute toggle.
7. **Migrate** meaningful content from prototype: `for-loop`, `while-loop`, `if/else`, `arrays` (and any other blocks already fully stepped in `TOPICS`).

---

## Tasks

### 2.1 Types and curriculum shape

- [ ] Add `src/types/curriculum.ts` (or similar): `Topic`, `GuidedExample`, `CodeSegment`, `CodeLine`, `ExecutionStep`.
- [ ] `ExecutionStep` must support: line id/index, optional `focusSegmentIds`, `variables`, optional `changed[]`, optional console lines, `explanation`.
- [ ] Export a `curriculum` array or modular `examples/*.ts` imported by `curriculum.ts`.

### 2.2 Code highlighting

- [ ] Server-side-ish **syntax tinting**: reuse approach from prototype (`highlight.ts`) but render **segments** as spans so step can add `data-segment-id` or class for focus.
- [ ] `CodeLine` component: line number + flex row of segments.
- [ ] Focus styles: default line highlight + stronger style for focused segment(s).

### 2.3 Playback engine

- [ ] `engine/playback.ts`: pure helpers (next/prev bounds, progress fraction).
- [ ] `hooks/usePlayback.ts`: state, `play`, `pause`, `stepForward`, `stepBack`, `reset`, `setSpeedMs`, `tick` for auto-advance.
- [ ] On last step auto-stop; optional completion sound + toast (match prototype behavior).

### 2.4 Side panels

- [ ] `VariableInspector`: show `typeof` for primitives; arrays as chips with `arrIdx` highlight when step provides it.
- [ ] `ConsolePanel`: render step’s accumulated output; empty state when not started.
- [ ] `ExplanationPanel`: HTML-safe or plain text; support short formatting if needed.

### 2.5 Controls and shortcuts

- [ ] Buttons: reset, back, play/pause, forward; speed slider; step counter `n / total`.
- [ ] Keyboard: Space (play/pause), ArrowLeft/Right (step), R (reset) — match prototype; ignore when focus in text input (Phase 03 editor).

### 2.6 Sound

- [ ] `hooks/useSound.ts` + `engine/sound.ts`: lazy `AudioContext`, enable after first user gesture if required.
- [ ] Cues: step, change, output, complete, back (align with prototype).

### 2.7 Content migration

- [ ] Move `TOPICS` from `init_example.html` into `src/data/examples/*.ts` with new types.
- [ ] Wire topic nav + example pills (or equivalent) to loaded examples.
- [ ] Keep **Learn** tab fully functional; **Practice** remains placeholder until Phase 03.

## Acceptance Criteria

| # | Criterion |
|---|-----------|
| A1 | Selecting a topic + example loads code and stepping works end-to-end. |
| A2 | At least one example demonstrates **segment** focus (e.g. `for` line split into init / condition / update steps). |
| A3 | Variables update per step; `changed` drives visual “flash” or border highlight. |
| A4 | Arrays show index highlight when step includes `arrIdx` (or equivalent field). |
| A5 | Console panel shows log history consistent with stepped `console` steps. |
| A6 | Play/pause, speed slider, forward/back, reset behave correctly; progress bar updates. |
| A7 | Sound toggle works; no uncaught errors if audio blocked. |
| A8 | All migrated examples from prototype run without data errors (line indices match segment model). |
| A9 | No business logic left as a single 1000-line HTML script; core is split into modules. |
